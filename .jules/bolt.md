## 2024-05-19 - Lennard-Jones Dimensional Error and Math Optimizations
**Learning:** The Lennard-Jones force calculations in `matlab/LJ_Force.m` and `matlab/doPotCalcLJ.m` were fundamentally mathematically broken, missing a `1/r` normalizer when multiplying scalar force by distance vector. Furthermore, MATLAB explicitly calculates distances with `sqrt()` and exponentiation `^13` / `^7` which is extremely slow. Fixing the math algorithm to use the physically correct dimensional derivative naturally removes `sqrt()`, allowing optimization entirely via squared distances and multiplicative inverses.
**Action:** Always verify if a performance bottleneck like `sqrt()` and high exponentiation in physics inner-loops is due to incorrect mathematical formulations. If fixing the algorithm to its physically correct dimension allows bypassing the bottleneck, do so.
